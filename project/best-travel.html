Best Travel旅游平台

项目介绍：

为⾹港Best Travel旅⾏社的导游提供个性化定制旅游⾏程的系统，系统分为前后台，前台为导游 ⾃定义⽣成⾏程并⽣成报价单等功能，后台功能提供交通，景点，饮⻝等模块的基础数据⽀撑，接⼊Visa等 ⽀付服务商，最终能⽣成⼀个旅⾏社内部使⽤的⾃定义旅游⾏程的⽐价平台。

项目难点分析及解决方案：

1. 项目需求不明确，只提供了一个旧版的系统让我随机发挥，一边说取其精华去其糟粕，说简单能懂就好，完成一版后说怎么和之前不一样，感谢你聪明可爱的客户。

2. 需要给用户提供一个自定义生成旅游报价单的功能，自定义行程的功能类似一个加入购物车的操作，只不过是多了区分日期的操作。这里引入了redis来操作界面，把临时的行程数据存放到redis中，并给redis的key加上一个过期时间，当key过期这个行程自动删除掉，避免总是频繁有没有生成行程的单据保留在内存中。

3. 需要防止行程单重复提交，把几个具有唯一属性的值来生成一个hash值，用redis判断这个值是否存在，如果存在就不允许重复提交（我在这里也是顺手加上了过期时间来判断行程单是否失效，这个失效的时间是要根据后台设置的日期来设置的）

用了什么技术：

Java11 + IDEA + PostgreSQL + Spring Boot + Mybatis-plus + Redis + Vue

干了什么：

与客户沟通，对需求进⾏拆解整理，与设计团队进⾏沟通制定流程，参与完整产品原型设计
