Payment Gateway支付网关

项目介绍：

公司的各个网站需要同时接入多个支付服务商，原有功能设计出现了大量冗余代码，需要重复部署多套代码，由此开发出一个整合多个支付服务商配置的平台，公司内部人员只需要通过一个后台进行不同支付服务商的基础配置，前后端通过若依低代码平台并通过一些调整适配，业务上的设计，单独完成项目。

用了什么技术：

Java11 + IDEA + SQL Server + MyBatis-plus + RuoYi

干了什么：

刚入职的时候没活，看到公司以前的支付api，是用.net做的，代码耦合度高，每次需要调用这个api都要另外部署一个，所以将支付模块单独拆分出一个服务让其他服务通过http来请求。
